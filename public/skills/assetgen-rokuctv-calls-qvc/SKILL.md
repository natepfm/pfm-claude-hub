# AssetGen — Roku CTV Calls QVC / SaveMaxAuto Live (/ag.rctv.c.qvc)

The complete asset-gen job for a SaveMaxAuto Live (QVC-style home-shopping) Roku CTV Calls 16:9 creative. **Run ALL steps as one pipeline — the editor never chases individual steps.** Sibling of `/ag.rctv.c.bn`; shares its spine (backgrounding, streaming, digit-verify, additive DaVinci, delivery) but the format is different where it counts:

- **The format lives in the GRAPHICS.** Two hosts sell "today's deal" (the caller's policy) while a dense QVC-style graphics package — Deal Card, Savings Board, discount checklist, live caller counter — does the persuasion. When in doubt, MORE tasteful on-screen info, not less.
- **Graphics are OVERLAYS, never in-frame (Sam, locked 07-08).** Every QVC graphic is created SEPARATELY and laid over the Veo footage in the timeline. Veo frames stay CLEAN — set dressing only (navy display panels, set signage); no gen'd data graphics inside any video frame.
- **The hero mechanic is the live caller transformation:** caller states their rate → CONNECTING → the board ticks DOWN in lockstep with the discount checklist lighting up → caller returns with the new rate. The board sync happens in EDIT; asset gen delivers pixel-registered graphic states + host lines with the beats separated.
- **Numbers law (INVERTED from BN):** RATES are spoken (naturalized words) AND shown on the board — spoken value and board state must match exactly. The PHONE NUMBER is never spoken — screen-only, whole spot ("the number's on your screen"). Spoken rates are ATTEMPT-FIRST, not a hill to die on (Sam, 07-08): if a rate line won't pass the spoken-rate QC after ~2 v0N refires, swap to its de-numbered fallback (the board carries the number) and move on — never spend hours fighting Veo on a number.
- **Callers are never on video.** Phone-quality ElevenLabs VO + a caller lower-third. Hosts carry all camera time.

All 16:9 (CTV self-declares horizontal). Brand-clean: our own SaveMaxAuto Live look — never replicate QVC/HSN logos or trade dress. SMA brand → AI-performer disclaimer required on the finished cut (Rule 4; 16:9 one-line preset via `add-ai-disclaimer`).

## ⚡ Backgrounding rule
Every gen >30s runs `run_in_background: true`; 3+ gens in one action = always backgrounded. Never chain `cmd &` inside a backgrounded shell (orphans the fire). Stream every result the instant it lands (Rule 5) — 📲 + widget, BEFORE QC.

## The pipeline

**0. Stage** (via `stage-request` on an initial batch; iterations/refires fire direct). Two build shapes:
- **FIRST BUILD of the format** (e.g. Houston maiden): everything below runs — masters, set, graphic bases, full clip set.
- **CITY/STATE VARIANT of a built format (e.g. Seattle):** masters + set plate + ALL scene frames (incl. the LISTEN hold seeds) + the listening-hold library reuse 100% — holds have no dialogue and their audio is replaced by the real caller VO in edit. QVC host lines are city-saturated by design, so most host clips REGEN (same seeds, new dialogue); graphics regen is nearly free — template constants (city, anchor, counter, callers, phone) + rerun + `call-graphics` for the new tracking number. **Per-variant data that must come from the REQUEST, never carried over: the city anchor rate (must be a real market anchor), the tracking number, caller names.** BN variants are graphics-heavy; QVC variants are video-heavy.
- **BRAND VARIANT (e.g. SaveMaxHomes Houston):** everything a city variant touches PLUS: (a) compliance flips to the new vertical (Home Calls: $50/mo / $600/yr floor, "as low as $600/year", approved claims up to $815/yr / up to 80%); (b) the HERO DISCOUNT beat is vertical-specific — Auto's "homeowner discount" has no Home meaning; pick the Home hero (bundling / new roof / security) and re-derive the checklist items + Deal Card spec (dwelling coverage, a believable house, not a truck); (c) set plate gets ONE surgical edit swapping the easel sign to the real new-brand wordmark passed as a ref → re-approve → re-derive F1/F2/F3 → re-fire; hosts can stay; (d) the AI-performer disclaimer still applies — synthetic performers regardless of brand.

**1. Cast + set lock (first build only).**
- **Host masters** via `pfm-character-master` (gpt_image_2): Diane — seasoned home-shopping host, warm, 50s, polished jewel-tone blazer; Ryan — hype co-host, 30s, works the board. Wardrobe is LOCKED per creative — one live show, identical outfits in every frame (consistency, not the b-roll rotation rule). Save both to the Character Library + Prompts sheet (always, no ask).
- **Environment master (set plate)** — the EMPTY SaveMaxAuto Live set as its own locked master, gen'd + approved BEFORE any people go in. **Model: `gpt_image_2`, 16:9, 2k** (exact replication + text, not NB). Full spec — the locked branded PROP CLUSTER, the 🔴 three asset-gen laws (reference-first · real SMA wordmark as a 2nd ref, never prose · no edit-stacking haze), and the Houston approved plate + prompt paths — lives in `references/set-plate-notes.md`. Load it before any set-plate gen. Approved plate → library + Prompts sheet like a character master.
- **Scene frames** via NB Pro 16:9 — multi-cam coverage of ONE staging, never solos (Sam, 07-08: hosts are behind the desk the whole show; a "solo on set" reads as the partner vanishing): **F1** waist-up two-shot at the counter (masters + plate refs; faces large enough for lip-sync — the master staging, approve FIRST), then **F2/F3 = closer favor-angle cameras generated FROM the approved F1 as the single scene ref** — tight medium favoring one host, BOTH hosts in their exact F1 positions (Diane camera-left, Ryan camera-right), the partner's shoulder edge-visible, same axis. **Speaker law (Sam, 07-08): any frame or clip depicting speech names WHO is talking and who is NOT** — "Diane mid-word, mouth open; Ryan listening, mouth CLOSED, nodding" — never "both mid-sentence" (a both-talking seed frame breeds both-talking clips). Favor frames carry their host as the speaker (per-line seeds match speaker to frame). **Prop-consistency law (Sam, 07-08): every favor-angle names ALL of F1's desk props explicitly** — the locked prop cluster (SaveMaxAuto easel sign, Max plush + bobblehead + statue, corded desk phone, blue toy sedan, two paw mugs, boxes on the lower shelf, red "take a call" buzzer on the host's right), hosts at F1 spacing — the frames intercut in the edit, so a missing/flipped/moved prop between angles is a continuity break; NB drops unnamed props on reframes (F3 lost the phone and squeezed spacing across three takes until each was named). *(Old laptops-on-desk staging retired 07-09 — see the set-plate prop cluster above.)*
- Editor approves plate + frames before any Veo fire. Scene-frame law: every clip seeds from F1/F2/F3 (16:9 ref == 16:9 render), NEVER a bare master.

**1b. 🔴 SEED-AFFORDANCE LAW (Sam-driven diagnosis, 07-09): a blank white lightbox/panel in a seed frame IS a screen affordance — Veo will decorate it (hallucinated text, lights, full screens materializing) on nearly EVERY take, and no prompt negative/positive lock reliably beats it (4 locked takes = 4 decorations on Houston L09).** Fix at the SEED, not the prompt: gpt_image_2 single-change surgery removing the blank panel (continuous wall in its place) → refire off the surgical seed (Houston: `SMA_Live_F2_diane_favor_NOSCREEN.png`, validated clean first pull ×2). Generalize: audit every scene frame for blank-screen affordances BEFORE the batch; favor-angle frames that crop a lightbox large behind a host are the highest risk. Edit-side backstop for already-fired clips: static garbage matte from the seed still over the wall region (locked camera holds it perfectly).

**2a-0. 🔴 HOLD SEEDS ARE THEIR OWN SEEDS (Houston 07-10, 6 wasted clips).** The F2/F3 dialogue favor frames show their host MID-SPEECH — that's what makes them good dialogue seeds and poison for holds: an open mouth in a D-locked seed IS a talking affordance and Veo talks through every take, no prompt lock wins. Every angle needs a **LISTEN seed** (gpt_image_2 single-pass surgery from the APPROVED frame: mouth fully closed, hands quiet on the desk, any lightbox removed in the same pass; optional `_LISTEN_CAM` variant staring down the lens). **Model ladder for holds: LISTEN seed on Veo Lite (caller-audio recipe) → if a host still talks after ~3 takes, Kling 3.0** (`kling3_0 --duration 8`, prose prompt, mute any stray sound in edit — caller VO owns the channel). Houston result: F3 cleared on Veo with the LISTEN seed; F2 needed Kling. Gotcha: gpt kept putting Ryan's hand back ON the red button ("beside" refused twice) — hand-on-button is a press-then-speak affordance; remove the hand from frame if it matters.

**2a. Reaction/listening holds — the CALLER-AUDIO recipe (Sam-validated 07-09, supersedes both babble-fighting and no-audio).** Veo cannot hold silence — it fills dead air with host babble. Two tested escapes: (a) `--generate_audio false` (mouths stay shut, truly silent file — works, but sterile); (b) **the winner: audio ON, but REDIRECT the audio to a phone CALLER** — "a caller's voice heard through the desk telephone's speakerphone, thin compressed phone-line quality; NEITHER host speaks, the ONLY voice is the caller's through the phone." Veo voices the caller and the hosts *listen* naturally (discovered on R2_v01 which accidentally did this). Don't forbid the audio channel — give it a job that isn't the hosts. Fire holds D-locked (same seed `--start-image` AND `--end-image`), 3 flavors per angle (attentive / delighted / warm-glance) across F1/F2/F3 = a 9-clip listening library covering every caller beat and cutaway. Ref impl: `qvc_rlisten_fire.py` pattern (Houston: `Rlisten_F1a…F3c`).

**2. Host clips** — `veo3_1_lite --generate_audio true`, 16:9, 8s, count=1, frame_to_video from F1/F2/F3. Script→manifest law:
- One clip per host line or exchange; merge micro-lines into one clip the editor splits (same framing); ≤22 words per 8s clip.
- Two-shot clips: ONLY the assigned speaker talks — prompt "Diane speaks, mid-word; Ryan listens, mouth CLOSED, nodding" (never let Veo split a line across hosts; name the non-speaker's closed mouth explicitly — Sam, 07-08).
- Board-sync lines (the discount run) get SINGLES with the beats as short sentences — the editor cuts board states on her beats. No ellipses/dashes/ALL CAPS in dialogue; rates naturalized ("two hundred forty a month", "fifty-eight dollars a month"). Write a de-numbered fallback variant alongside every rate-bearing line at manifest time (used only if Veo fights the number — see QC).
- Caller beats on the host side = **D-locked reaction holds** (F1 UUID as BOTH `--start-image` and `--end-image`), no dialogue — "they listen to the caller, nodding, upbeat" + negatives "no talking, no lip movement, no dialogue".
- **Audio clarity block on every host clip** (carry the BN anti-muffle block): capture "CLOSE, DRY, CLEAN and CRISP by the studio broadcast microphone… every consonant crisp, the voice forward in the mix" + environment "quiet treated broadcast studio, zero room tone — the hosts bring the energy, not the room" + negatives "muffled voice, muddy audio, distant-sounding voice, low-pass filtered audio, roomy sound, echo, reverb, off-mic voice".
- NSFW false-positives are stochastic — refire with a v0N slug bump, dialogue untouched; refires never overwrite.

**2b. Refire craft — the Houston defect→fix map (07-09/07-10; check here BEFORE improvising a fix).** Refires fire direct (no gate), stream every take the instant it lands, ≥20 still preflights.
- **Hallucinated text/lights/screens on the background** → NOT a prompt problem — seed-affordance (law 1b). Surgery the seed, refire off it. Prompt-lock refires are wasted credits after ~2 tries.
- **Music cues creeping in** → add "no music anywhere in the clip" to the prompt AND consider restaging the line onto a different angle — a stubborn defect on a two-shot often vanishes on a favor-angle restage (Houston: L02→F2, L03b→F3 both cleared this way). A restage is a SCRIPT change: update master JSON + manifest + Notion same turn.
- **Button-press beats:** a line that starts with the call-button press is staged "clip OPENS on the red-button press, then the line" (L04/L12). A line that ENDS on the press is staged "line → press → HARD SILENCE, mouths closed to end of clip" (L03b) — without the hard-silence clause Veo keeps talking through the press.
- **Unscripted interjections/bark tails** ("Yeah", "Mm-hmm", dog bark) → Whisper-check ALL takes before refiring; often an earlier take is already clean (L07: v05 refire introduced a new interjection, v01/v03 were the clean reads).
- **Non-speaker lip movement** → the seed frame's speaker law was violated or the prompt dropped the "mouth CLOSED" clause; re-check both before firing.

**Rev-change law (feedback_request_script_single_canon):** ANY dialogue/staging change = master JSON + dialogue manifest + the Notion request's numbered Copy block updated IN PLACE, all in the SAME turn. The Copy block is the ONE current master script — house format `N. **ID — SPEAKER (angle):** dialogue *(staging)*`, numbered in show order, caller VO lines (M/N) inline at their beats. No tables, no stacked rev blocks, no change-log footers.

**3. Caller VO** — ElevenLabs (never Veo). Pick phone-natural voices via voice search (maiden cast: Marcus — easygoing American male 30s-40s; Linda — bright American female 50s), render the caller lines naturalized, then the **phone-line pass**:
`ffmpeg -i in.mp3 -af "highpass=f=300,lowpass=f=3400,acompressor=threshold=-20dB:ratio=3:attack=5:release=60,alimiter=limit=0.9" -ar 48000 out_phone.wav`
Deliver BOTH raw + `_phone` versions to `Elements/Audio/VO/`. Whisper-verify the spoken rates.

**4. Graphics package — THE HEAVY LIFT. Owned by `qvc-screen-graphics` — invoke it; do not build graphics inside this skill.** Method (v3, Sam-approved 07-09): the whole overlay package is **HTML/CSS rendered by headless Chrome** to 1920×1080 RGBA PNGs — real typography, exact digits, native alpha, zero credits (the v1 gpt-base + PIL-state method was REJECTED as "cheap"; that record lives in the graphics skill's `references/method-history.md`). States are template params on ONE layout — pixel-registered ticks for free. Then the **animation pass** (same skill): PIL keyframe engine → ProRes 4444 **alpha MOVs**, each entrance/tick/pop its own short clip so the editor places drops on dialogue beats.

Package shape (Houston = 18 PNGs + 18 MOVs): Deal Panel rate states **+ a blank IDLE state**, discount checklist tick states **+ a blank IDLE** (the checklist ENTERS blank — never with a pre-checked item), Order Bar counter states, caller lower-thirds (nowrap — long name+city must never wrap), city stat pill (e.g. HOUSTON AVG $235/MO for the anchor beat), CONNECTING, channel bug w/ the real wordmark. **IDLE law (Sam, 07-09): every persistent panel needs a blank version for no-caller stretches** — the board can't show a caller's numbers before/between callers; animate IDLE→value as a crossfade tick. Banner + end card (tracking number) stay with `call-graphics`.

Dense, specific detail is the POINT (the spec card reads as real because it lists everything) — fill every Deal Card line from the request; believable local vehicle per the request text (text only — no logos).

**5. QC** — local/in-chat fires are NEVER auto-QC'd. After delivery, OFFER QC via a plain multi-choice ask (run audio-qc / run visual-qc / skip); AGF/mini runs keep mandatory QC. RECOMMEND the **spoken-rate compliance check** in the offer as a pre-delivery step: every clip containing a dollar amount Whisper-verifies the exact naturalized number (a wrong spoken rate is a compliance kill, not a retake); a rate line still failing after ~2 v0N refires → fire its de-numbered fallback variant instead of grinding Veo (the board carries the number). **Digit-verify every graphic state regardless** (Read each PNG — part of delivery, not the QC offer): tracking number digit-for-digit, SMA.Insure spelling, every $ value against the script's board cues, checklist order == script order.

**6. Compliance frame (bake into every variant):** $50/mo Auto Calls floor — every shown/spoken rate above it; caller counter is SOCIAL PROOF (count of people), never scarcity; the ONLY urgency is "lines close at 5 PM today"; anchor rate must be a real market anchor from the request (never invented retail value); aggregate savings claim stated separately ("nearly $X a year on average" phrasing from the request); "could save" framing everywhere else.

**7. Naming** — city + concept in EVERY filename, number at the end of graphics for cross-check: `qvc_live_roku_calls_houston_L07_diane_board_v01.mp4`, `qvc_live_roku_calls_houston_R1_react_v01.mp4`, `Savings Board - Houston - 240 - GPT_xxxx.png`, `Deal Card - Houston now58 (713) 936-4745 - GPT_xxxx.png`, `Caller LT - MARCUS Houston - PIL.png`, `marcus_M1_phone.wav`.

**8. Deliver** — stream as they land; DaVinci **first import = whole-folder mirror; later adds = ADDITIVE ONLY** (never DeleteFolders, never touch Creatives/); Notion `✅ Assets Generated` comment + Asset Gen lifecycle; full 📁/🔗/🦊 handoff; **SMA disclaimer flag** for the final cut (add-ai-disclaimer, 16:9 one-line preset).

## Assembly — Palmier bare spine (v0 DRAFT, maiden practice 07.08.26 — pending Sam's sign-off)

The QVC creative assembles as a **show rundown**: host lines in manifest order with the reaction holds sitting exactly where the caller VOs land. Raw spine only — graphics/captions/b-roll are the edit.

1. **Order:** L01→L04 · **R1** · L05→L08 · **R2** · L09→L12 · **R3** · L13→L15 (from the dialogue manifest, never re-derived).
2. **Speed from the runtime spec:** holds run 1.0× (they carry caller VO); solve host speed from `(spoken_total)/(target − hold_total)`, cap at house 1.3×. Houston: 101.8s spoken, 90s spec → 1.3×.
3. **Flat pipeline stands:** pre-speed hosts to `Elements/Footage/Primary/Presped/` (`setpts+atempo`, fps 24), build at 1.0.
4. **Native trim (v0.6.3+):** place hosts UNTRIMMED → `remove_silence` once (Silero VAD: head breaths + tails + mid-line pauses, ripple-closed) → **THEN** `insert_clips` the holds (VAD would eat them as dead air) → mute hold audio (v0.6.3: audio id rides on the video clip as `audio.id`). Hold placeholders: R1=6s (caller line + CONNECTING), R2/R3=4s.
5. **Canvas law:** `add_clips` auto-matches the canvas to the first clip (720p!) — re-assert 1920×1080@24 after placement, transforms `{1.0, 0.999}` AFTER the assert, read back.
6. **Transcript QC gate before showing:** `get_transcript` → per-line diff vs the manifest (order, completeness, extra speech, spoken rates). Maiden catches: L03b unscripted babble tail (trim, don't refire), VAD can clip a soft first word (L14 "Our"), holds can carry muted speech-bleed (R2 — never unmute raw), Veo interjection bleed on board-run lines (editor's call).
7. L15's audio stays live in the spine (stripped under the end card in edit).
